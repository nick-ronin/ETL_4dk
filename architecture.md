# Архитектура проекта

## Краткое описание

Проект реализован по модульной архитектуре и представляет собой сервис обработки данных, поступающих из различных источников в форматах CSV и Excel. Каждый этап обработки вынесен в отдельный модуль, что позволяет независимо развивать, тестировать и использовать компоненты системы. Все модули взаимодействуют последовательно, передавая результаты обработки друг другу.

---

# Общая архитектура

Архитектура проекта построена по принципу последовательного конвейера обработки данных (Pipeline). Каждый модуль выполняет одну задачу и передаёт результат следующему этапу обработки.

Основной поток обработки начинается с получения файла через API, после чего данные проходят загрузку, приведение структуры к единому формату, нормализацию, поиск дубликатов, оценку качества и формирование итоговых результатов.

---

# Структура проекта

Проект разделён на независимые модули, каждый из которых отвечает за отдельный этап обработки данных.

- **api** — принимает запросы пользователей и запускает процесс обработки файлов.
- **source_processing** — выполняет загрузку файлов и приведение структуры данных к единому виду.
- **normalizer** — очищает и нормализует значения полей.
- **deduplicator** — выполняет поиск частичных и полных дубликатов.
- **quality** — рассчитывает показатели качества обработанных данных.
- **exporter** — сохраняет результаты обработки в Excel и формирует служебные отчёты.
- **logger** — ведёт журнал выполнения операций.
- **tests** — содержит автоматические тесты проекта.
- **docs** — содержит документацию проекта.

# Основные компоненты

### API

Является точкой входа в систему. Принимает файлы от пользователя, запускает обработку данных и возвращает результат в формате JSON или ZIP-архива с Excel-файлами.

[Документация к API](api/api.md)

### Source Processing

Состоит из модулей Source Loader и Source Mapper. Выполняет чтение файлов, определение структуры таблицы, сопоставление названий колонок и формирование единого набора полей для дальнейшей обработки.

[Документация к source processing](service/source_processing/source_processing.md)

### Normalizer

Приводит значения полей к единому формату. Выполняется очистка ИНН, телефонов, адресов электронной почты, адресов и названий организаций.

[Документация к Normalizer](service/normalizer/normalization.md)

### Deduplicator

Выполняет поиск частичных и полных дубликатов по заданным полям и формирует отчёт о найденных совпадениях.

[Документация к Deduplicator](service/deduplicator/deduplication.md)

### Quality

Анализирует качество подготовленных данных и рассчитывает основные показатели полноты, уникальности, точности и согласованности.

[Документация к quality](service/quality/quality.md)

### Exporter

Формирует итоговые Excel-файлы с обработанными данными и создаёт служебный отчёт о выполненной обработке.

[Документация к exporter](service/exporter/exporter.md)

# Взаимодействие компонентов

Все компоненты работают последовательно и образуют единый конвейер обработки данных.

```text
Пользователь
      │
      ▼
     API
      │
      ▼
Source Loader
      │
      ▼
Source Mapper
      │
      ▼
Normalizer
      │
      ▼
Deduplicator
      │
      ▼
Quality
      │
      ▼
Exporter
      │
      ▼
Результат (JSON или Excel)
```

Каждый модуль получает данные от предыдущего этапа, выполняет собственную обработку и передаёт результат следующему модулю.

---

# Поток обработки данных

После загрузки файла через API выполняется чтение содержимого и определение структуры данных. Затем происходит сопоставление колонок с внутренней моделью проекта. После формирования единой структуры данные проходят этап нормализации, где очищаются значения основных полей. Далее выполняется поиск частичных и полных дубликатов, после чего рассчитываются показатели качества данных. Завершающим этапом является формирование итогового результата, который возвращается пользователю в формате JSON или в виде Excel-файлов.

---

# Взаимосвязь модулей

Модули проекта взаимодействуют между собой через объекты `DataFrame` и словари с результатами обработки.

- API запускает весь процесс обработки.
- Source Loader передаёт считанные данные в Source Mapper.
- Source Mapper формирует единую структуру данных для Normalizer.
- Normalizer передаёт очищенные данные в Deduplicator.
- Deduplicator передаёт информацию о найденных дубликатах модулю Quality.
- Quality рассчитывает показатели качества обработанных данных.
- Exporter получает итоговый набор данных и формирует выходные файлы.

# Принципы архитектуры

При разработке проекта использовались следующие принципы:

- модульность компонентов;
- разделение ответственности между модулями;
- последовательная обработка данных;
- использование единого формата передачи данных (`pandas.DataFrame`);
- независимость модулей друг от друга;
- возможность расширения существующего функционала без изменения архитектуры проекта;
- повторное использование компонентов в различных сценариях обработки.

---

# Расширяемость проекта

Архитектура проекта позволяет добавлять новые этапы обработки без изменения существующих модулей. Для расширения функциональности достаточно реализовать новый сервис, подключить его в общий конвейер обработки и при необходимости добавить документацию и тесты. Благодаря модульной организации проекта новые компоненты могут использоваться независимо или включаться в общий процесс обработки данных.
