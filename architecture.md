# Архитектура проекта

## Общий принцип работы

Обработка данных построена в виде последовательного pipeline:

1. Пользователь загружает исходный CSV/XLSX файл через API.
2. Файл передается в модуль загрузки и подготовки источника.
3. Выполняется проверка структуры входных данных.
4. Данные приводятся к единой структуре колонок.
5. Выполняется нормализация значений.
6. Проводится поиск совпадений и дубликатов.
7. Результаты сохраняются в Excel.


## Структура компонентов

### API

API является точкой входа в систему.

Отвечает за:
- прием файлов;
- запуск процесса обработки;
- управление последовательностью выполнения этапов;


### Source Processing

Модуль отвечает за работу с исходными файлами.

Включает:
- загрузку файлов;
- обработку структуры источника;
- сопоставление колонок различных форматов.

Цель модуля — привести данные из разных источников к единому представлению перед дальнейшей обработкой.


### Normalizer

Модуль нормализации отвечает за приведение значений к единому формату.

Используется после подготовки структуры данных и перед поиском совпадений.

Подробное описание методов очистки находится в файле `normalization.md`.


### Deduplicator

Модуль поиска коллизий используется после нормализации данных.

Его задача — обнаружить записи, которые могут относиться к одной сущности.

Подробное описание алгоритма поиска дубликатов находится в файле `deduplication.md`.

## Формирование результата

После завершения всех этапов обработки API формирует итоговый результат.

Результат включает:
- обработанные данные в формате JSON;
- Excel-файл с найденными коллизиями;
- сводную информацию о количестве обработанных строк и найденных совпадениях.

Сохранение результатов выполняется в процессе работы API и отдельных методов обработки.

## Взаимодействие модулей

Модули работают последовательно:

'API → Source Processing → Normalizer → Deduplicator → Формирование результата'

Каждый этап получает результат предыдущего этапа и передает подготовленные данные дальше.
