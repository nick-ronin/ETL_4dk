# ETL_check

## Описание проекта

ETL_4dk — сервис обработки данных из CSV/XLSX файлов. Проект предназначен для очистки, нормализации и подготовки данных из разных источников перед их дальнейшим использованием.

Основная задача сервиса — привести данные к единому формату, чтобы записи из разных источников можно было корректно сравнивать, объединять и создать удобный файл для использования

## Доступ к сервису

Развёрнутый экземпляр проекта доступен по следующим адресам:

- **API:** https://db.vodalchuk.ru/
- **Swagger:** https://db.vodalchuk.ru/docs

## Возможности проекта

Сервис выполняет полный цикл обработки входных файлов:

- загрузка CSV/XLSX файлов — [Source Loader](service/source_loader/source_loader.md);
- проверка структуры и обязательных полей — [Validator](service/validator/validator.md);
- сопоставление колонок разных источников — [Source Mapper](service/source_mapper/source_mapper.md);
- очистка и нормализация данных — [Normalizer](service/normalizer/normalizer.md);
- поиск совпадений и дубликатов — [Deduplicator](service/deduplicator/deduplication.md);
- расчёт показателей качества данных — [Quality](service/quality/quality.md);
- формирование Excel-отчётов и ZIP-архива — [Exporter](service/exporter/exporter.md);
- обработка запросов и запуск ETL-пайплайна — [API](api/api.md).

## Правила обработки данных

Перед использованием данные проходят несколько этапов подготовки.

На этапе загрузки выполняется проверка структуры входного файла и сопоставление исходных колонок с необходимыми полями.

При нормализации данные приводятся к единому формату:
- удаляются лишние пробелы, переносы строк и некорректные символы;
- названия компаний приводятся к виду, пригодному для сравнения между источниками;
- адреса приводятся к единому формату;
- телефоны и email очищаются и стандартизируются;
- идентификаторы (например, ИНН) проверяются и приводятся к корректному виду.

После обработки выполняется проверка совпадений между записями по ключевым полям. Найденные коллизии сохраняются в отчёты для последующего анализа.

Подробные правила обработки отдельных полей описаны в документации модулей нормализации и дедупликации.
