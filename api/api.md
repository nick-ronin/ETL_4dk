# API загрузки файлов

Модуль реализует API для обработки CSV/XLSX файлов. После получения файла выполняется полный ETL-процесс: загрузка, приведение структуры к единому формату, нормализация данных, поиск дубликатов, оценка качества и формирование результата. В зависимости от параметра запроса API возвращает JSON с обработанными данными либо ZIP-архив с Excel-отчётами.

---

# Назначение

API предназначен для автоматической обработки данных, поступающих из различных источников.

Модуль позволяет:

- принимать CSV, XLS и XLSX файлы;
- приводить структуру данных к единому формату;
- выполнять очистку и нормализацию полей;
- находить частичные и полные дубликаты;
- рассчитывать показатели качества данных;
- формировать итоговые Excel-отчёты;
- возвращать результат обработки в формате JSON или ZIP-архива.

---

# Входные данные

Метод принимает HTTP POST-запрос.

### Endpoint

```http
POST /upload/
```

### Параметры

| Параметр | Тип | Описание |
|----------|-----|----------|
| `file` | UploadFile | CSV, XLS или XLSX файл |
| `download` | bool | `false` — вернуть JSON, `true` — вернуть ZIP с отчётами |

Пример запроса:

```http
POST /upload/?download=true
```

---

# Процесс работы

После получения файла API последовательно выполняет следующие этапы.

### Шаг 1. Загрузка файла

Модуль Source Loader:

- определяет тип файла;
- считывает данные;
- загружает идентификаторы как строки;
- формирует DataFrame.

↓

### Шаг 2. Сопоставление структуры

Модуль Source Mapper:

- сопоставляет названия колонок;
- удаляет лишние поля;
- формирует единую структуру данных.

↓

### Шаг 3. Проверка данных (Validator)

Модуль Validator:

проверяет наличие обязательных колонок;
определяет, можно ли продолжить обработку файла;
формирует информацию о результате проверки .

↓

### Шаг 4. Нормализация

Выполняется очистка:

- ИНН;
- телефонов;
- email;
- адресов;
- названий компаний.

Также создаются служебные признаки:

- `has_email`;
- `has_phone`.

↓

### Шаг 5. Поиск дубликатов

Модуль дедупликации:

- ищет совпадения по ИНН;
- email;
- телефону;
- названию компании;
- определяет полные дубликаты.

↓

### Шаг 6. Оценка качества данных

Рассчитываются показатели:

- Completeness;
- Accuracy;
- Uniqueness;
- Consistency.

После чего вычисляется итоговый Quality Score.

↓

### Шаг 7. Формирование результата

Для каждой записи добавляются служебные поля:

- `id`;
- `source_name`;
- `source_type`;
- `source_date`.

↓

### Шаг 8. Возврат результата

Если

```text
download = false
```

возвращается JSON.

Если

```text
download = true
```

формируется ZIP-архив, содержащий:

- очищенный Excel-файл;
- отчёт обработки;
- отчёт по найденным дубликатам.

---

# Основные методы

### `upload_file()`

Основной endpoint обработки файла.

Принимает файл, запускает полный ETL-процесс и возвращает результат.

---

### `clean_json_value()`

Рекурсивно заменяет значения:

- NaN;
- Infinity;
- -Infinity

на `None`, чтобы данные могли быть корректно сериализованы в JSON.

---

# Формат результата

## JSON

Возвращается при:

```text
download=false
```

Пример:

```json
{
  "status": "OK",
  "rows": 154,
  "columns": [
    "short_name",
    "inn",
    "phone",
    "email",
    "address"
  ],
  "quality": {
    "overall_quality_score": 94.6
  },
  "data": [
    {
      "short_name": "Ромашка",
      "inn": "7701234567"
    }
  ]
}
```

---

## ZIP-архив

Возвращается при:

```text
download=true
```

Архив содержит:

```
cleaned_data.xlsx
processing_report.xlsx
collisions.xlsx
```

---

# Пример работы

Запрос:

```http
POST /upload/?download=false
```

↓

Загрузка файла:

```
companies.xlsx
```

↓

Выполняется:

```
Source Loader
      ↓
Source Mapper
      ↓
Validator
      ↓
Normalization
      ↓
Deduplication
      ↓
Quality Score
      ↓
JSON
```

При запросе

```http
POST /upload/?download=true
```

результатом станет ZIP-архив с Excel-отчётами.

---

# Использование в проекте

Модуль является центральной точкой входа в систему обработки данных.

Во время работы он использует:

- `source_loader` — загрузка файла;
- `source_mapper` — сопоставление структуры;
- 'validator' — проверка обязательных полей;
- `normalizer` — очистка данных;
- `deduplicator` — поиск дубликатов;
- `quality` — расчёт качества данных;
- `exporter` — формирование Excel-отчётов.

# Возможные ошибки

| Ошибка | Причина |
|----------|----------|
| 400 Bad Request | Некорректный формат файла |
| 400 Bad Request | Ошибка загрузки Source Loader |
| 400 Bad Request | Ошибка сопоставления структуры |
| 500 Internal Server Error | Ошибка формирования Excel |
| 500 Internal Server Error | Не удалось сформировать ZIP-архив |
| FileNotFoundError | Временный файл отсутствует |
| HTTPException | Ошибка обработки данных |
