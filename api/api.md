# API загрузки файлов

## Назначение

API предназначен для загрузки файлов с данными о компаниях в форматах CSV и Excel. Основная задача API — принять исходный файл, обработать данные перед загрузкой в базу данных, привести информацию из разных источников к единому формату и подготовить данные для дальнейшего объединения.

API реализует полный процесс обработки данных: загрузку файла, проверку структуры и преобразование колонок.


## Endpoint

Для загрузки файла используется метод:

POST /upload/

Метод принимает файл через параметр `file`.

Поддерживаемые форматы:

- CSV;
- XLSX;
- XLS.


После получения файла он сохраняется во временное хранилище и передаётся на дальнейшую обработку.


## Процесс обработки данных

### Загрузка файла

На первом этапе выполняется загрузка исходного файла.

Модуль SourceLoader отвечает за:

- чтение CSV или Excel файла;
- определение структуры таблицы;
- поиск необходимых колонок;
- приведение названий колонок к общему виду.

Результатом этапа является подготовленный файл для дальнейшей обработки.


### Проверка данных

После загрузки выполняется проверка входных данных.

Проверяется:

- наличие обязательных колонок;
- корректность структуры файла;
- возможность дальнейшей обработки данных.

Если файл не соответствует требованиям, API возвращает ошибку.

Затем файл обрабатывается, происходит дедупликация и нормализация данных.

## Ответ API

После успешной обработки API возвращает JSON с результатами.

Ответ содержит:

status — статус выполнения обработки.

validation — результат проверки входного файла.

rows — количество обработанных строк.

columns — список колонок после обработки.

data — очищенные и подготовленные данные.


Пример ответа:

{
    "status": "SUCCESS",
    "validation": "OK",
    "rows": 100,
    "columns": [
        "short_name",
        "inn",
        "phone",
        "email",
        "address"
    ],
    "data": []
}


## Обработка ошибок

API возвращает ошибку в следующих случаях:

- файл имеет неподдерживаемый формат;
- отсутствуют обязательные поля;
- невозможно выполнить обработку данных;
- произошла внутренняя ошибка сервера.


Основные коды ошибок:

400 — ошибка входных данных.

500 — внутренняя ошибка обработки.
