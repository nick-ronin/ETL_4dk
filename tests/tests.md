# Документация по тестированию проекта

Модуль содержит автоматические тесты, проверяющие корректность работы основных компонентов проекта. Тестирование выполняется с использованием библиотеки `pytest` и позволяет убедиться, что загрузка, обработка, дедупликация, оценка качества данных и экспорт результатов работают корректно.

---

# Назначение

Тесты предназначены для проверки работоспособности основных модулей проекта и предотвращения появления ошибок после внесения изменений в код.

Покрываются следующие компоненты:

- Source Loader;
- Source Mapper;
- Дедупликация;
- Оценка качества данных;
- Экспорт результатов.

---

# Входные данные

В качестве входных данных используются:

- тестовые `DataFrame`;
- временные Excel-файлы;
- временные директории (`tmp_path`);
- словари с результатами предыдущих этапов обработки.

Все тестовые данные создаются автоматически и не требуют использования реальных файлов проекта.

---

# Процесс работы

Во время выполнения тестов для каждого метода создаются входные данные, после чего вызывается тестируемая функция. Полученный результат сравнивается с ожидаемым значением при помощи операторов `assert`.

Тесты проверяют:

- корректность обработки входных данных;
- правильность структуры результата;
- создание необходимых файлов;
- обработку пустых данных;
- обработку ошибочных ситуаций.

---

# Основные тесты

## Source Loader

Проверяются:

- сопоставление названий колонок;
- нормализация заголовков;
- проверка обязательных полей.

### Проверяемые методы

- `map_columns()`
- `normalize_columns()`
- `validate_required_columns()`

---

## Source Mapper

Проверяются:

- удаление лишних колонок;
- добавление отсутствующих обязательных полей;
- обработка Excel-файлов.

### Проверяемые методы

- `filter_columns()`
- `process_file()`

---

## Дедупликация

Проверяются:

- отсутствие дубликатов;
- поиск частичных дубликатов;
- поиск полных дубликатов;
- работа с пустыми значениями;
- обработка пустого DataFrame.

### Проверяемый метод

- `get_duplicates()`

---

## Оценка качества данных

Проверяются:

- диапазон итоговой оценки качества;
- наличие всех метрик;
- обработка пустых данных;
- расчёт качества без обязательных колонок;
- обработка полностью заполненных данных;
- влияние найденных дубликатов.

### Проверяемый метод

- `calculate_data_quality_score()`

---

## Экспорт данных

Проверяются:

- сохранение Excel-файла;
- простой экспорт;
- экспорт после Source Mapper;
- обработка ошибок;
- обработка пустого DataFrame.

### Проверяемые методы

- `export_to_excel()`
- `export_simple()`
- `export_from_mapper()`

---

# Формат результата

При успешном выполнении все тесты завершаются без ошибок.

Пример запуска:

```bash
pytest
```

Результат:

```text
==================== test session starts ====================

tests/test_loader.py ........
tests/test_mapper.py .......
tests/test_quality.py .......
tests/test_duplicates.py ......
tests/test_exporter.py ......

==================== 31 passed ====================
```

При обнаружении ошибки `pytest` выводит название теста и описание причины сбоя.

---

# Пример работы

Запуск одного файла тестов:

```bash
pytest tests/test_quality.py
```

Запуск одного теста:

```bash
pytest tests/test_quality.py::test_quality_perfect_data
```

Запуск всех тестов:

```bash
pytest
```

---

# Использование в проекте

Тесты рекомендуется запускать:

- после изменения бизнес-логики;
- после добавления новых функций;
- перед публикацией новой версии проекта;
- после исправления ошибок.

Регулярное выполнение тестов позволяет убедиться, что изменения не нарушили работу существующего функционала.
