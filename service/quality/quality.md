# Quality Score

Модуль `Quality Score` предназначен для оценки качества обработанных данных. После завершения нормализации и дедупликации он рассчитывает четыре основные метрики качества: полноту, уникальность, точность и согласованность, а затем формирует итоговую оценку (Overall Quality Score).

---

# Назначение

Модуль используется для автоматической оценки качества данных после их обработки.

Основные задачи:

- оценка полноты заполнения обязательных полей;
- определение уровня уникальности данных;
- проверка корректности заполнения ключевых полей;
- оценка согласованности форматов данных;
- вычисление итогового показателя качества данных.

---

# Входные данные

Основной метод принимает два параметра.

| Параметр | Тип | Описание |
|----------|-----|----------|
| `df` | `DataFrame` | Обработанные данные после нормализации |
| `dedup_result` | `dict` | Результат работы модуля дедупликации |

Пример:

```python
quality_report = calculate_data_quality_score(
    df,
    dedup_result
)
```

---

# Процесс работы

После получения данных модуль выполняет несколько последовательных этапов.

### 1. Расчёт полноты (Completeness)

Проверяется заполненность обязательных колонок из `MVP_COLUMNS`.

Оценивается доля заполненных ячеек относительно общего количества обязательных полей.

---

### 2. Расчёт уникальности (Uniqueness)

Используются результаты дедупликации.

Метрика рассчитывается по количеству строк, участвующих в коллизиях.

Чем меньше найдено дубликатов, тем выше показатель уникальности.

---

### 3. Расчёт точности (Accuracy)

Проверяются ключевые поля:

- `inn`
- `phone`
- `email`
- `address`

Оценивается доля непустых значений после выполнения нормализации.

---

### 4. Расчёт согласованности (Consistency)

Для полей:

- `inn`
- `phone`
- `email`

анализируется единообразие формата данных.

Проверяется соответствие длины значений ожидаемым диапазонам.

---

### 5. Итоговая оценка

После вычисления всех метрик рассчитывается общий показатель качества данных:

```text
Overall Quality Score =
(Completeness +
Uniqueness +
Accuracy +
Consistency) / 4
```

---

# Основные методы

### `calculate_data_quality_score()`

Основной метод модуля.

Выполняет расчёт всех метрик качества и формирует итоговый отчёт.

---

# Формат результата

Метод возвращает словарь следующего вида:

```python
{
    "total_rows": 150,

    "metrics": {
        "completeness": 0.97,
        "uniqueness": 0.95,
        "accuracy": 0.98,
        "consistency": 1.00
    },

    "overall_quality_score": 0.98,

    "details": {
        "duplicate_rows_affected": 8,
        "checked_columns": [
            "inn",
            "phone",
            "email",
            "address"
        ]
    }
}
```

Если входной DataFrame пуст:

```python
{
    "error": "Пустой датасет",
    "overall_quality_score": 0
}
```

# Пример работы

```python
from service.quality.quality import calculate_data_quality_score

quality_report = calculate_data_quality_score(
    df,
    dedup_result
)

print(quality_report["overall_quality_score"])
```

Результат:

```text
Overall Quality Score: 0.96
```

# Использование в проекте

Модуль используется после завершения основных этапов обработки данных.

Полученная оценка качества включается в JSON-ответ API и позволяет оценить качество обработанного набора данных перед дальнейшим использованием.

---

# Возможные ошибки

| Ошибка | Причина |
|----------|----------|
| `Пустой датасет` | Передан пустой DataFrame |
| `KeyError` | Отсутствует ожидаемая колонка в данных |
| `AttributeError` | Передан объект, не являющийся DataFrame |
| `TypeError` | Некорректный тип входных параметров |
| `overall_quality_score = 0` | Не удалось выполнить расчёт качества из-за отсутствия данных |
