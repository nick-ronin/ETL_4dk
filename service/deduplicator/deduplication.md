# Дедупликация данных

## Назначение

Модуль дедупликации предназначен для поиска повторяющихся записей в данных перед загрузкой информации в базу данных.

Так как данные поступают из разных источников, одна и та же компания может встречаться несколько раз с одинаковыми или частично совпадающими значениями. Дедупликация помогает выявить такие записи для дальнейшего анализа и объединения.

## Метод поиска дубликатов

### `get_duplicates()`

Метод принимает таблицу данных (`DataFrame`) и список колонок для проверки.

В процессе работы метод:

- ищет совпадения по отдельным полям;
- ищет полностью одинаковые строки;
- сохраняет номера строк исходного файла;
- формирует отчёт о найденных дубликатах.


## Виды дубликатов


### Частичные дубликаты

Определяются по отдельным колонкам.

Например, одинаковый ИНН или одинаковый email в нескольких записях.

Используются для поиска возможных совпадений компаний.


### Полные дубликаты

Определяются, когда все значения строки полностью совпадают.

Такие записи считаются повторными и могут быть удалены или объединены.


## Формирование отчёта

После проверки создаётся Excel-файл с результатами.

Отчёт содержит:

### Лист "Коллизии"

Все найденные совпадения с указанием:

- источника данных;
- номера строки;
- поля, по которому найден дубль;
- значения дубликата;
- типа совпадения.


### Лист "Сводка"

Содержит общую информацию о количестве найденных групп дубликатов и затронутых строках.


### Лист "Полные_дубликаты"

Содержит только полностью совпадающие записи.


## Использование в ETL-процессе

Модуль используется перед загрузкой данных в базу данных.

Результаты дедупликации помогают:

- найти повторные записи;
- объединить данные из разных источников;
- уменьшить количество дублей в БД;
- повысить качество итоговой информации.


## Результат работы

Метод возвращает:

- путь к Excel-отчёту с найденными совпадениями;
- сводную информацию по дубликатам;
- количество строк, участвующих в коллизиях.
