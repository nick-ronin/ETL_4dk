# Дедупликация данных

## Назначение

Модуль предназначен для поиска повторяющихся записей в обработанных данных и формирования отчёта о найденных коллизиях.

Проверка выполняется как по отдельным полям, так и по полному совпадению строк. Результатом работы является Excel-файл с подробной информацией о найденных дубликатах и краткой сводкой.

---

## Входные данные

Метод `get_duplicates()` принимает:

| Параметр | Описание |
|----------|----------|
| `df` | DataFrame с обработанными данными |
| `columns` | Список колонок, по которым выполняется поиск частичных дубликатов |
| `source_name` | Имя исходного файла (используется в отчёте) |
| `output_dir` | Папка для сохранения отчёта |

---

## Процесс работы

Работа метода состоит из нескольких этапов.

1. Создаётся копия исходной таблицы и добавляется колонка с номерами строк исходного файла.

2. Выполняется поиск частичных дубликатов по указанным колонкам.

3. Выполняется поиск полностью одинаковых строк.

4. Для каждого найденного совпадения формируется группа дубликатов с указанием:
   - источника данных;
   - номера строки;
   - типа дубликата;
   - поля, по которому найдено совпадение;
   - значения дубликата.

5. Создаётся сводная информация о количестве найденных групп и строк.

6. Формируется Excel-отчёт.

---

## Основной метод

### `get_duplicates()`

**Назначение**

Выполняет поиск частичных и полных дубликатов и формирует Excel-отчёт.

**Принимает**

- таблицу данных (`DataFrame`);
- список колонок для поиска частичных совпадений;
- имя исходного файла;
- папку для сохранения результата.

**Возвращает**

```python
{
    "collisions_file": "output/collisions_file.xlsx",
    "summary": {...},
    "rows_affected": 25
}
```

где:

- `collisions_file` — путь к сформированному отчёту;
- `summary` — статистика по найденным дубликатам;
- `rows_affected` — количество строк, участвующих в коллизиях.

---

## Формат результата

После завершения работы создаётся Excel-файл, содержащий три листа.

### Лист «Коллизии»

Содержит все найденные совпадения.

Для каждой записи указываются:

- Источник;
- Номер строки;
- Группа дубликата;
- Тип дубликата;
- Поле;
- Значение дубликата;
- остальные данные строки.

---

### Лист «Сводка»

Содержит информацию по каждому проверяемому полю:

- количество найденных групп;
- количество строк, участвующих в коллизиях.

---

### Лист «Полные_дубликаты»

Содержит только полностью совпадающие записи.

---

## Пример работы

### Исходные данные

| short_name | inn | email |
|------------|-----|----------------|
| Ромашка | 7701234567 | info@test.ru |
| Альфа | 7701234567 | alfa@test.ru |
| Ромашка | 7709876543 | info@test.ru |

### Результат

Будут найдены две группы частичных дубликатов:

- по полю `inn`;
- по полю `email`.

Если две строки полностью совпадают по всем значениям, они также будут добавлены в раздел полных дубликатов.

---

## Использование в проекте

Модуль используется после завершения нормализации данных.

Результаты дедупликации используются для анализа качества данных и поиска повторяющихся записей перед дальнейшей обработкой.

---

## Возможные ошибки

| Причина | Результат |
|----------|-----------|
| Отсутствует указанная колонка | Колонка пропускается |
| Дубликаты отсутствуют | Создаётся отчёт с сообщением «Коллизии не найдены» |
| Ошибка записи файла | Excel-файл не создаётся |
