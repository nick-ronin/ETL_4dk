# File Cleaner – ротация выходных файлов

## Назначение

Модуль `file_cleaner` предназначен для автоматической очистки служебных директорий от старых файлов.

После каждой успешной обработки он оставляет только последние **N** файлов (по времени последнего изменения), удаляя более старые. Это предотвращает бесконтрольное накопление файлов и переполнение дискового пространства.

По умолчанию используется для следующих директорий:

- `output/cleaned_data`
- `output/collisions`
- `output/processing_report`
- `output/log`

---

## Основная функция

```python
from service.file_cleaner.cleaner import rotate_files

rotate_files(
    directory="output/log",
    max_files=11,
    reserved=["app.log"]
)
```

### Параметры

| Параметр | Тип | Описание |
|----------|-----|----------|
| `directory` | `str` | Путь к директории, в которой необходимо выполнить ротацию файлов. |
| `max_files` | `int` | Максимальное количество файлов, которое должно оставаться в директории. |
| `reserved` | `list[str]`, *опционально* | Список файлов, которые никогда не удаляются (например, `["app.log"]`). |

### Возвращаемое значение

Функция ничего не возвращает (`None`).

Если указанная директория отсутствует, выполнение завершается без возникновения исключений.

---

## Алгоритм работы

Функция выполняет следующие действия:

1. Проверяет существование указанной директории.
2. Получает список всех файлов в директории.
3. Исключает из обработки файлы, указанные в параметре `reserved`.
4. Сортирует оставшиеся файлы по времени последнего изменения (от самых старых к самым новым).
5. Пока количество файлов превышает значение `max_files`, удаляет самый старый файл.
6. Если удалить конкретный файл не удалось (например, он используется другим процессом), ошибка игнорируется, и обработка продолжается для остальных файлов.

---

## Пример использования

### Ротация очищенных данных

```python
save_data(df, output_folder="output/cleaned_data")

rotate_files(
    directory="output/cleaned_data",
    max_files=10
)
```

### Ротация логов

Файл `app.log` всегда сохраняется независимо от количества остальных файлов.

```python
rotate_files(
    directory="output/log",
    max_files=11,
    reserved=["app.log"]
)
```

## Использование в пайплайне

Вызовы `rotate_files()` встроены в основной процесс обработки (`api/upload.py`).

| Директория | `max_files` | `reserved` | Момент вызова |
|------------|------------:|------------|---------------|
| `output/collisions` | 10 | — | После дедупликации |
| `output/cleaned_data` | 10 | — | После экспорта очищенных данных |
| `output/processing_report` | 10 | — | После сохранения отчёта |
| `output/log` | 11 | `app.log` | После успешного завершения обработки |

---

## Особенности

- Ротация выполняется автоматически после успешной обработки файла.
- Удаляются только самые старые файлы.
- Зарезервированные файлы никогда не удаляются.
- Отсутствие директории не считается ошибкой.
- Ошибка удаления отдельного файла не прерывает выполнение функции.
