# Exporter

Модуль `Exporter` предназначен для сохранения обработанных данных в Excel-файлы. Помимо основного файла с данными, модуль может сформировать служебный отчёт с информацией о выполненной обработке. Используется на завершающем этапе ETL-процесса после очистки, нормализации, дедупликации и оценки качества данных.

---

# Назначение

Модуль выполняет экспорт результатов обработки в формат Excel.

Основные задачи:

- сохранение обработанного DataFrame в Excel;
- создание папки для сохранения файлов при необходимости;
- автоматическое добавление временной метки к имени файла;
- формирование служебного отчёта об обработке;
- подготовка файлов для последующей выдачи пользователю через API.

---

# Входные данные

Большинство методов принимают объект `pandas.DataFrame`, содержащий обработанные данные.

Основные параметры:

| Параметр | Тип | Описание |
|----------|-----|----------|
| `df` | `DataFrame` | Обработанные данные |
| `output_folder` | `str` | Папка для сохранения файлов |
| `filename` | `str` | Имя создаваемого Excel-файла |
| `filename_main` | `str` | Имя основного файла |
| `filename_report` | `str` | Имя файла отчёта |
| `mapper_result` | `dict` | Результат работы `source_mapper` |

---

# Процесс работы

После получения обработанных данных модуль выполняет следующие действия:

1. Проверяет существование папки для сохранения файлов.
2. Создаёт временную метку.
3. Формирует имя выходного файла.
4. Сохраняет DataFrame в Excel.
5. При необходимости формирует служебный отчёт.
6. Возвращает информацию о результате экспорта.

---

# Основные методы

### `get_timestamp()`

Возвращает текущую дату и время в формате:

```text
YYYYMMDD_HHMMSS
```

Используется при формировании уникальных имён файлов.

---

### `ensure_folder()`

Проверяет существование папки и создаёт её при необходимости.

---

### `export_to_excel()`

Экспортирует DataFrame в один Excel-файл.

Возвращает информацию о сохранённом файле.

---

### `export_with_report()`

Экспортирует данные и дополнительно создаёт Excel-отчёт с информацией о выгрузке.

Отчёт содержит:

- имя файла;
- дату обработки;
- количество строк;
- количество колонок;
- список колонок.

---

### `export_from_mapper()`

Принимает результат работы `Source Mapper`, извлекает DataFrame и выполняет экспорт.

---

### `export_simple()`

Выполняет упрощённый экспорт одного DataFrame без формирования отчёта.

Возвращает путь к созданному файлу.

---

# Формат результата

### `export_to_excel()`

Возвращает словарь:

```python
{
    "status": "OK",
    "file_path": "output/standard_clients_20260710_153045.xlsx",
    "timestamp": "20260710_153045",
    "rows": 250,
    "columns": 18
}
```

---

### `export_with_report()`

Возвращает:

```python
{
    "status": "OK",
    "main_file": "output/standard_clients_20260710_153045.xlsx",
    "report_file": "output/processing_report_20260710_153045.xlsx",
    "rows": 250,
    "columns": 18
}
```

---

### `export_simple()`

Возвращает путь к созданному файлу:

```text
output/data_export_20260710_153045.xlsx
```

---

# Пример работы

```python
from service.exporter.exporter import export_with_report

result = export_with_report(
    df=df,
    output_folder="output",
    filename_main="cleaned_data",
    filename_report="processing_report"
)

if result["status"] == "OK":
    print(result["main_file"])
    print(result["report_file"])
```

После выполнения будут созданы файлы:

```text
output/
├── cleaned_data_20260710_153045.xlsx
└── processing_report_20260710_153045.xlsx
```

---

# Использование в проекте

Модуль используется на завершающем этапе обработки данных.

Полученные Excel-файлы затем используются API для формирования ZIP-архива и передачи пользователю.

# 8. Возможные ошибки

| Ошибка | Причина |
|----------|----------|
| `DataFrame пуст` | Переданы пустые данные |
| `PermissionError` | Выходной файл открыт в Excel |
| `FileNotFoundError` | Недоступна папка сохранения |
| `ERROR` | Ошибка при записи Excel-файла |
| `ERROR` | Отсутствуют данные после `source_mapper` |
| `OSError` | Недостаточно прав на запись в папку |
