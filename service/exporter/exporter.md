# Exporter

Модуль `Exporter` предназначен для сохранения результатов обработки данных и формирования итоговых файлов. Помимо сохранения очищенных данных в Excel, модуль создаёт текстовый отчёт с информацией о выполненной обработке и журналом работы приложения. Используется на завершающем этапе ETL-процесса после очистки, нормализации, дедупликации и оценки качества данных.

---

# Назначение

Модуль выполняет экспорт результатов обработки данных.

Основные задачи:

- сохранение обработанного DataFrame в Excel;
- формирование текстового отчёта о выполненной обработке;
- добавление сводной информации о качестве данных, дубликатах и валидации;
- включение полного журнала обработки в отчёт;
- автоматическое создание папки для сохранения файлов;
- автоматическое добавление временной метки к именам файлов;
- подготовка файлов для последующей выдачи пользователю через API.

---

# Входные данные

Методы модуля принимают результаты работы предыдущих этапов ETL-процесса.

Основные параметры:

| Параметр | Тип | Описание |
|----------|-----|----------|
| `df` | `DataFrame` | Обработанные данные |
| `filename` | `str` | Имя исходного файла |
| `date` | `str` | Дата обработки |
| `rows` | `int` | Количество обработанных строк |
| `mapper_result` | `dict` | Результат работы Source Mapper |
| `quality` | `dict` | Результат расчёта Quality Score |
| `dedup` | `dict` | Результат поиска дубликатов |
| `log_file_path` | `str` | Путь к журналу обработки |
| `output_folder` | `str` | Папка для сохранения файлов |
| `base_name` | `str` | Базовое имя создаваемого файла |

---

# Процесс работы

После получения результатов обработки модуль выполняет следующие действия:

1. Формирует сводную информацию об обработке.
2. Добавляет сведения о прохождении валидации.
3. Добавляет информацию о найденных дубликатах.
4. Добавляет показатели качества данных.
5. Считывает журнал обработки из файла логов.
6. Формирует единый текстовый отчёт.
7. Создаёт папку для сохранения файлов при необходимости.
8. Сохраняет очищенные данные в Excel.
9. Сохраняет текстовый отчёт.
10. Возвращает информацию о созданных файлах.

# Основные методы

### `build_summary_text()`

Формирует текст итогового отчёта.

Отчёт включает:

- информацию об исходном файле;
- дату обработки;
- количество строк;
- результат проверки обязательных полей;
- количество найденных дубликатов;
- показатели качества данных;
- полный журнал обработки приложения.

Возвращает строку с готовым содержимым отчёта.

---

### `save_report()`

Сохраняет сформированный текстовый отчёт в файл формата `.txt`.

Возвращает информацию о сохранённом файле.

---

### `save_data()`

Сохраняет обработанный DataFrame в Excel-файл.

Возвращает информацию о результате сохранения.

---

```python
    rows=120,
    mapper_result=mapper_result,

```

# Использование в проекте

Модуль используется на завершающем этапе ETL-процесса.

Получает результаты работы следующих модулей:

- Source Mapper;
- Validator;
- Deduplicator;
- Quality;
- Logger.

На основании полученных данных формирует:

- Excel-файл с обработанными данными;
- текстовый отчёт о выполненной обработке.

Сформированные файлы затем используются API для передачи пользователю.

# Возможные ошибки

| Ошибка | Причина |
|----------|----------|
| `FileNotFoundError` | Не найден файл журнала обработки |
| `PermissionError` | Нет прав на запись файлов |
| `OSError` | Ошибка файловой системы при создании папки или записи файла |
| `ERROR` | Ошибка сохранения Excel-файла |
| `ERROR` | Ошибка сохранения текстового отчёта |
| `UnicodeEncodeError` | Ошибка записи текста в отчёт |
