## 📋 Назначение

Модуль `source_loader` приводит сырые файлы (выгрузки из 1С, Excel-отчёты, CSV-файлы) к **единому стандарту** для дальнейшей обработки (скоринг, поиск дублей, анализ данных).

## 🚀 Как использовать

### Вариант 1: Запуск через терминал (для тестирования)

```bash
python src/source_loader.py
```

Программа автоматически:
1. Найдёт все Excel и CSV файлы в папке `test_files`
2. Обработает каждый файл
3. Сохранит результаты в папку `output`

### Вариант 2: Использование в коде

```python
from src.source_loader import process_uploaded_file

# Обработка одного файла
result = process_uploaded_file(
    file_path="test_files/ваш_файл.xlsx",
    required_columns=['inn'],      # Обязательные колонки
    save_output=True,               # Сохранять ли результат
    output_folder="output"          # Папка для сохранения
)

# Получение результатов
print(f"Статус: {result['status']}")          # OK / WARNING / ERROR
print(f"Строк: {len(result['df'])}")          # Количество строк
print(f"Сохранён: {result['saved_file']}")    # Путь к файлу

# Доступ к данным
df = result['df']  # pandas DataFrame с нормализованными данными
```

### Вариант 3: Интеграция с FastAPI

```python
from fastapi import FastAPI, UploadFile, File
import shutil
from src.source_loader import process_uploaded_file

app = FastAPI()

@app.post("/upload")
async def upload_file(file: UploadFile = File(...)):
    # Сохраняем загруженный файл
    file_path = f"temp_{file.filename}"
    with open(file_path, "wb") as buffer:
        shutil.copyfileobj(file.file, buffer)
    
    # Запускаем обработку
    result = process_uploaded_file(
        file_path,
        required_columns=['inn'],
        save_output=True
    )
    
    return {
        "status": result['status'],
        "saved_file": result['saved_file'],
        "rows": len(result['df']) if result['df'] is not None else 0
    }
```

### Входные данные

| Формат | Расширение | Примечание |

| Excel | `.xlsx`, `.xls` | Поддерживается автоматически |

- **Имя:** `standard_<исходное_имя>_<дата_время>.xlsx`

- **Содержание:** Данные с колонками, приведёнными к единому стандарту

#### 2. Словарь с результатом (при использовании в коде)


## 🔧 Как работает маппинг (сопоставление колонок)

Модуль использует словарь `COLUMN_MAPPING`, который сопоставляет различные варианты названий колонок с единым стандартом.

### Пример

Исходные названия → Стандартное название:

| Исходное название | Стандартное название |
|-------------------|---------------------|
| `ИНН`, `инн`, `ИНН организации` | `inn` |
| `Телефон`, `Мобильный телефон`, `Телефоны` | `phone` |
| `Email`, `E-mail`, `Почта` | `email` |
| `Полное наименование`, `Название организации` | `full_name` |

### Как добавить новый вариант названия

Открой файл `src/source_loader.py`, найди словарь `COLUMN_MAPPING` и добавь новый вариант:

```python
'phone': ['Телефон', 'Мобильный телефон', 'Новый вариант названия'],
```

## 🧪 Стандарт полей (ERD + ТЗ)

Модуль поддерживает все поля из:

1. **ERD (расширенный список)** — для работы с финансовыми показателями, адресами, контактами и т.д.
2. **Стандарта контрагента из ТЗ** — для совместимости со скоринговыми командами

### Основные стандартные колонки

| Категория | Поля |
|-----------|------|
| **Идентификаторы** | `inn`, `kpp`, `ogrn`, `okpo`, `okato`, `oktmo`, `okfs` |
| **Названия** | `short_name`, `full_name`, `company_name`, `company_name_normalized` |
| **Контакты** | `email`, `phone`, `contact_phone_mobile`, `contact_website`, `contact_whatsapp`, `contact_telegram`, `contact_vkontakte` |
| **Адреса** | `address`, `region`, `city`, `postal_index`, `district` |
| **Финансы** | `revenue`, `net_profit`, `receivables`, `payables`, `inventory`, `fixed_assets` |
| **Регистрация** | `registration_date`, `authorized_capital`, `employee_count`, `msp_category` |
| **Руководитель** | `director_full_name`, `contact_person`, `position`, `director_inn` |
| **Служебные** | `source_name`, `source_type`, `source_date`, `has_email`, `has_phone`, `duplicate_flag`, `data_quality_score` |

**Полный список** всех поддерживаемых полей можно найти в словаре `COLUMN_MAPPING` в файле `source_loader.py`.

## 🛠️ Параметры функции `process_uploaded_file()`

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `file_path` | `str` | **Обязательный** | Путь к файлу на диске |
| `required_columns` | `list` | `['inn']` | Список обязательных колонок. Если колонка отсутствует → статус `WARNING` |
| `save_output` | `bool` | `True` | Сохранять ли результат в Excel |
| `output_folder` | `str` | `'output'` | Папка для сохранения результата |

---

## 📈 Статусы обработки

| Статус | Описание |
|--------|----------|
| `OK` | Файл загружен, все обязательные колонки найдены |
| `WARNING` | Файл загружен, но некоторые обязательные колонки отсутствуют |
| `ERROR` | Ошибка при загрузке или обработке файла |

---

## 🐛 Частые ошибки и их решение

| Ошибка | Решение |
|--------|---------|
| `ModuleNotFoundError: No module named 'pandas'` | Установи библиотеку: `pip install pandas openpyxl` |
| `FileNotFoundError: Файл не найден` | Проверь путь к файлу. Файл должен существовать |
| `Missing optional dependency 'openpyxl'` | Установи openpyxl: `pip install openpyxl` |
| `Неподдерживаемый формат` | Используй файлы с расширением `.xlsx`, `.xls` или `.csv` |
| `ВНИМАНИЕ! Отсутствуют колонки: inn` | В файле нет колонки с ИНН. Проверь название колонки или добавь в словарь |

## 📝 Как расширить словарь маппинга

Если в твоих файлах встречаются новые названия колонок:

1. Открой `src/source_loader.py`
2. Найди словарь `COLUMN_MAPPING`
3. Добавь новый вариант в нужный список:

```python
'phone': ['Телефон', 'Мобильный телефон', 'Новое название телефона'],
```

4. Сохрани файл и запусти заново

## 👥 Кто использует этот модуль

| Модуль | Что делает | Как использует source_loader |
|--------|------------|------------------------------|
| **normalizer** | Очищает значения (ИНН, телефон, email) | Получает DataFrame с нормализованными колонками |
| **deduplicator** | Находит дубли по ИНН, телефону, email | Работает с колонками `inn`, `phone`, `email` |
| **data_quality** | Считает качество данных | Анализирует заполненность полей |
| **exporter** | Формирует итоговые файлы | Сохраняет результат в Excel |
| **FastAPI** | Веб-интерфейс для загрузки файлов | Вызывает `process_uploaded_file()` |
