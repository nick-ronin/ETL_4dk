# Source Loader

Модуль загрузки исходных файлов.

Source Loader является первым этапом обработки данных. Его задача — корректно считать CSV или Excel-файл, определить необходимые параметры чтения и вернуть данные в виде `DataFrame` для последующей обработки. Модуль не изменяет содержимое данных, а только обеспечивает их безопасную загрузку.

---

# Назначение

Модуль предназначен для загрузки исходных файлов и подготовки данных к дальнейшей обработке.

В процессе работы выполняется:

- проверка существования файла;
- определение формата файла;
- чтение файлов форматов CSV, XLS и XLSX;
- автоматическое определение кодировки CSV;
- чтение идентификаторов (ИНН, КПП, ОГРН и других) как строк для сохранения ведущих нулей;
- формирование результата в формате, удобном для последующих этапов ETL.

---

# Входные данные

На вход модуль получает путь к файлу.

Дополнительно используются настройки из `source_mapper`, определяющие:

- список колонок, которые необходимо читать как строки;
- варианты названий этих колонок.

---

# Процесс работы

После получения пути к файлу модуль выполняет несколько последовательных этапов.

1. Проверяет существование файла.

2. Определяет его формат.

3. Считывает только заголовки таблицы.

4. Определяет, какие столбцы должны иметь строковый тип данных.

5. Загружает файл с использованием найденных типов.

6. Возвращает результат обработки либо информацию об ошибке.

Содержимое файла на этом этапе не изменяется.

---

# Основные методы

## `_build_string_variants()`

Формирует множество возможных названий колонок, которые должны считываться как строки.

Использует настройки из `source_mapper`.

---

## `find_test_files(folder_path="test_files")`

Ищет все CSV и Excel-файлы в указанной папке.

Используется во время тестирования проекта.

Возвращает список найденных файлов.

---

## `load_file(file_path)`

Основной метод чтения файлов.

Выполняет:

- определение типа файла;
- чтение заголовков;
- определение строковых колонок;
- загрузку данных;
- возврат объекта `DataFrame`.

---

## `process_uploaded_file(file_path)`

Используется в основном пайплайне.

Вызывает `load_file()` и возвращает результат в едином формате.

Возвращаемая структура содержит:

- статус обработки;
- загруженный DataFrame;
- информацию об ошибке;
- сведения о прочитанных колонках.

---

# Формат результата

При успешной загрузке возвращается словарь:

```python
{
    "status": "OK",
    "df": DataFrame,
    "error": None,
    "mapping_info": {
        "total_columns": 25,
        "original_columns": [
            "ИНН",
            "Название",
            "Телефон"
        ]
    }
}
```

При возникновении ошибки:

```python
{
    "status": "ERROR",
    "df": None,
    "error": "Файл не найден",
    "mapping_info": None
}
```

# Пример работы

Исходный файл:

| ИНН | Название | Телефон |
|------|-----------|----------|
| 0123456789 | Компания А | 89001234567 |

После загрузки:

- файл успешно прочитан;
- ИНН сохранён как строка;
- DataFrame передан следующему этапу обработки.

---

# Использование в проекте

Source Loader является первым модулем обработки данных.

После успешной загрузки результат передаётся в:

- Source Mapper — для сопоставления колонок;
- модуль проверки структуры данных;
- последующие этапы нормализации и дедупликации.

---

# Возможные ошибки

Во время работы могут возникнуть следующие ошибки.

| Ошибка | Причина |
|---------|----------|
| FileNotFoundError | Указанный файл отсутствует |
| ValueError | Неподдерживаемый формат файла |
| UnicodeDecodeError | Ошибка определения кодировки CSV |
| Exception | Ошибка чтения файла |
