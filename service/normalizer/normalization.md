# Нормализация данных

## Назначение модуля

Модуль нормализации предназначен для очистки и приведения данных из различных источников к единому формату перед объединением таблиц и загрузкой информации в базу данных.

Так как данные поступают из разных CSV-файлов, одна и та же информация может быть записана по-разному. Например, название компании может содержать организационно-правовую форму, адреса могут иметь разные обозначения улиц и домов, а контактные данные могут содержать лишние символы.

Основная задача нормализации — привести значения к общему виду, чтобы данные из разных источников можно было сравнивать между собой и объединять записи одной компании.


## Принцип работы

Нормализация является этапом обработки данных в ETL-процессе.

Каждый метод нормализации принимает значение одного поля, выполняет очистку и возвращает подготовленное значение.


# Нормализация названия компании

## Метод

`clean_company_name()`

Метод используется для подготовки названий компаний к сравнению между различными источниками.

## Выполняет обработку

Метод:

- переводит значение в нижний регистр;
- заменяет букву "ё" на "е";
- удаляет кавычки;
- удаляет организационно-правовые формы (ООО, АО, ЗАО, ПАО и другие);
- удаляет лишние символы;
- удаляет лишние пробелы.


## Результат

До:

ООО "Ромашка"


После:

ромашка


Полученное значение используется для поиска совпадений между таблицами.


# Нормализация адреса

## Метод

`clean_address()`

Метод приводит адреса из разных источников к единому формату для дальнейшего сравнения.

## Выполняет обработку

Метод:

- удаляет переносы строк;
- удаляет лишние пробелы;
- переводит текст в нижний регистр;
- удаляет обозначения улиц (ул., улица, проспект и другие);
- приводит обозначения дома, корпуса и строения к единому формату;
- удаляет лишние запятые.


## Результат

До:

ул. Воронцовская, дом 6 строение 1


После:

воронцовская, 6 ст1


Такой формат позволяет использовать адрес для сопоставления данных из разных источников.


# Нормализация телефона

## Метод

`clean_phone()`

Метод используется для приведения телефонных номеров к единому формату.

## Выполняет обработку

Метод:

- удаляет все лишние символы;
- оставляет только цифры;
- заменяет российские номера, начинающиеся с 8, на формат +7;
- добавляет международный код;
- объединяет несколько номеров в одном поле;
- удаляет повторяющиеся значения.


## Результат

До:

8 (999) 123-45-67


После:

+79991234567


# Нормализация электронной почты

## Метод

`clean_email()`

Метод очищает email-адреса перед загрузкой в базу данных.

## Выполняет обработку

Метод:

- переводит email в нижний регистр;
- удаляет лишние пробелы;
- ищет корректные email с помощью регулярного выражения;
- поддерживает несколько адресов в одной строке;
- обрабатывает разделители (, ; / |);
- удаляет дубликаты.


## Результат

До:

 Test@mail.ru; INFO@mail.ru


После:

test@mail.ru, info@mail.ru


# Нормализация ИНН

## Метод

`clean_INN()`

Метод используется для очистки и проверки ИНН.


## Выполняет обработку

Метод:

- удаляет все символы кроме цифр;
- проверяет длину значения;
- проверяет соответствие формату ИНН.


## Результат

До:

123-456-7890


После:

1234567890


# Использование нормализованных данных

После выполнения очистки значения используются для объединения данных из разных источников.

Основные поля для сравнения:

- название компании;
- ИНН;
- адрес;
- телефон;
- электронная почта.


Нормализация позволяет уменьшить количество ошибок при объединении данных и повысить качество итоговой базы данных.
